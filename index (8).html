HOW IT
WORKS

01 — FETCH

🌐

Document Retrieval

Pulls 3 docs each from Wikipedia and arXiv using LangChain loaders.

02 — CHUNK

✂️

Text Splitting

Splits into 1000-char chunks with 200-char overlap via RecursiveCharacterTextSplitter.

03 — EMBED

🧠

Vector Encoding

Encodes chunks into 384-dim vectors via all-MiniLM-L6-v2, stored in ChromaDB.

04 — FORGE

⚡

RAG Assembly

Each heading independently queries the vector store, scores and deduplicates globally.

// project
SLIDE
FORGE
Drop a topic. Get a deck. Powered by RAG.
SlideForge is a full-stack RAG pipeline that automatically creates structured presentations from any topic. It fetches real documents from Wikipedia and arXiv, chunks and embeds them into a ChromaDB vector store, then retrieves and scores content per slide heading to produce factual, deduplicated bullet points — all served via FastAPI with PPTX and PDF export.
